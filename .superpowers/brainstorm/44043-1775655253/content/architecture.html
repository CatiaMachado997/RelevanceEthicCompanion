ESL permission model (progressive trust)

How EC asks for trust — once per action type
FIRST TIME — EC asks
"Create a GitHub issue? Allow once / Always allow / Never"
TRUSTED — auto-executes
Stored in tool_permissions. User can revoke anytime in Settings.
HIGH-RISK — always confirms
Send email, delete, post publicly — ESL enforces preview regardless of trust.

New database tables

Schema additions
tool_definitions -- the catalogue
  id, name, description, source_type
  auth_type (oauth|mcp|apikey)
  actions[] JSONB -- read_events, create_issue…
  risk_level (low|medium|high) per action
user_tool_connections -- per user
  user_id, tool_id, enabled
  credentials JSONB encrypted
  mcp_url (nullable — MCP only)
  connected_at, last_used_at
tool_permissions -- trust memory
  user_id, tool_id, action_name
  trust_level (ask|allow|deny)
  granted_at, expires_at (nullable)
