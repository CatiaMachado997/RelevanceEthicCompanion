What kind of tool marketplace?

Ethic Companion needs to know what model you're imagining for how users connect external tools

A

Curated catalogue

EC ships a growing list of pre-built integrations (Notion, Linear, Slack, GitHub, Todoist…). Users browse and toggle them on. You control quality — each connector is purpose-built and tested. Think Zapier's app directory or Claude's built-in tools.

Pros

  • Reliable, well-tested
  • Consistent UX
  • Easy to add ethical guardrails

Cons

  • You build every connector
  • Slow to expand
B

MCP / open protocol

Users point EC at any MCP server (Model Context Protocol — the open standard Claude uses). Thousands of community-built tools instantly available. Users paste a URL or run a local server.

Pros

  • Massive ecosystem now
  • Zero connector code for you
  • Standard protocol

Cons

  • Security surface is wide
  • ESL needs careful gating
  • Quality varies
C

Both — catalogue + MCP escape hatch

A curated catalogue for common tools (Google, Notion, Linear…), plus an advanced option to connect any MCP server for power users. Best of both worlds — great out-of-the-box, unlimited ceiling.

Pros

  • Safe defaults + power-user ceiling
  • Grow catalogue over time
  • MCP gives you the ecosystem

Cons

  • More to build initially
  • Two code paths to maintain
